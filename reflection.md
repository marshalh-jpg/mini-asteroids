This experience has been the smoothly agentic driven development I've experienced. The loop methodology is suprisingly effective. Even a basic loop script, such as the one present in this repo showed a surprising ability to progress through a complicated task in a truncated systematic way. Although, I'm sure this flow would be more robust if given a test framework to provide additional certainty that each step in the loop produced adequate results. Another interesting strategy I employed for this assignment was requiring the model to produce documentation in adherence to ADS-STE100 Simplified Technical English. To be honest, I've never read this specification myself, but I saw online a user mention that it significantly improved not only the readability of documentation produced, but also the ability for the llm to efficiently parse the language itself has written. While the language seems overly robotic in places, its undeniably concise which could make a large difference at scale. One last subject to mention is the advantage planning. Having the llm generate a plan before generating code seems to be a great way of tempering the eagerness of most models to jump into a problem without first considering better alternatives. An entire prompt and session dedicated to the formulation of just the plan contributed to the ease of development found in this assignment.
